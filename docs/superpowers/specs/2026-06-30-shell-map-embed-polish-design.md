# Shell 项目地图嵌入体验优化设计

日期：2026-06-30  
状态：已获用户口头确认，待书面审阅  
目标页面：`maps/?operator=Shell`

## 1. 目标

在不改变数据口径、项目内容和现有视觉方向的前提下，优化 Shell 项目地图在简道云中嵌入和宽嵌入场景下的可读性与交互体验。

本轮必须完成：

- 删除左下角的“点位为国家级聚合，不代表项目坐标”。
- 保留“圆点数字代表项目数量”。
- 保持现有小圆点、颜色、字体尺寸、项目详情和国家级聚合逻辑。
- 优化宽而矮窗口中的区域聚焦、控件避让、项目抽屉、搜索反馈和键盘操作。
- 先提供本地预览；用户确认后再更新 GitHub。

## 2. 非目标与约束

- 不修改 `maps/data.js`，不改变 Shell 51 个项目及其国家分布。
- 不增加或猜测项目级经纬度。
- 不引入资产数据，不开发多 Operator 数据源。
- 不重新设计品牌视觉，不缩小项目名称或详情字号。
- 不修改仓库根目录已有的公司卡片首页和横幅页面。
- 不在本轮替换 Leaflet、CARTO 或 CDN 架构。
- 简道云画布中地图下方的空白由宿主页面布局控制，不通过地图代码伪造高度修复。

## 3. 方案选择

采用“嵌入体验精修”方案：保留当前设计语言，在现有 HTML、CSS 和原生 JavaScript 结构内做小范围、可验证的改动。

未采用的方案：

- 只删除文字：风险最低，但不能解决宽嵌入视野过散和抽屉遮挡。
- 同时重构依赖和多 Operator 架构：范围过大，会延迟当前 Shell 试点验收。

## 4. 页面结构与视觉行为

### 4.1 左下角图例

图例只保留圆点符号和“圆点数字代表项目数量”。删除分隔线和国家级聚合说明。

移动端样式不能再使用“隐藏最后一个 `span`”的宽泛选择器，因为删除说明后，它会误隐藏唯一保留的文字。移动端仅隐藏已经不存在或明确标记的分隔元素，保留图例文字。

### 4.2 区域总览

总览仍以当前 Operator 的有效国家中心点为依据，不加入项目坐标。

默认使用国家点位边界计算视野。对于宽高比大于等于 3 的宽而矮容器，允许提高最小缩放级别，使东南亚成为视觉主体；应用候选缩放后必须验证所有国家圆点仍在安全可视区域内，否则退回标准边界缩放。

安全可视区域需要避开：

- 左上角标题卡；
- 右上角状态和缩放控件；
- 左下角图例；
- 右下角“区域总览”按钮。

窗口尺寸变化时重新计算总览视野，但不能在用户正在查看某个国家项目时强制跳回总览。

### 4.3 操作提示

“悬停查看数量 / 点击国家圆点展开项目”不再长期位于地图正中央。它改为靠近底部中央的轻量提示：首次加载后短暂显示，约 4 秒后淡出；用户点击、拖动或缩放后立即隐藏。点击“区域总览”不重复弹出该提示。

### 4.4 响应式布局

- 宽屏（大于等于 1100px）：保留右侧 460px 项目抽屉。
- 中等宽度（760–1099px）：右侧抽屉最大宽度收敛到约 420px，项目字号保持不变，为地图保留更多可见空间。
- 小屏（小于 760px）：继续使用底部面板，保留现有项目字号和详情单列布局。

标题、状态、图例和地图控件在各断点下不得互相遮挡。

## 5. 交互设计

### 5.1 打开国家项目抽屉

点击国家圆点后：

1. 打开对应国家项目列表。
2. 清空旧搜索条件并渲染完整列表。
3. 将选中圆点设为激活态。
4. 根据实际抽屉尺寸调整地图中心，使选中圆点位于抽屉之外的可见地图区域。

桌面端按抽屉实际宽度进行水平像素偏移；移动端按底部面板实际高度进行垂直像素偏移。不得继续使用固定的 `0.14` 经度偏移，因为该偏移与容器尺寸和抽屉宽度无关。

国家圆点只保留一条点击处理链，避免 Leaflet 事件和 DOM 事件重复触发同一次打开动画。

### 5.2 关闭与返回

- 关闭按钮维持现有行为。
- 按 `Esc` 可关闭抽屉。
- 关闭后清除圆点激活态，并把键盘焦点返回触发抽屉的国家圆点。
- “区域总览”关闭抽屉并重新适配所有国家点位。

### 5.3 项目列表与搜索

- 未搜索时显示国家项目总数。
- 输入搜索词后，摘要显示“匹配 X / 总计 Y”。
- 无匹配结果时保留明确提示，不改变原数据。
- 项目行展开和收起时同步更新 `aria-expanded`。
- 增加清晰的 `:focus-visible` 样式，不改变鼠标用户的现有视觉。

## 6. 数据流和组件边界

数据流保持不变：

1. `maps/data.js` 提供 `window.PROJECT_MAP_DATA`。
2. `maps/app.js` 根据 `?operator=` 筛选项目。
3. 项目按国家聚合，并使用现有 `countryConfig` 国家中心点创建圆点。
4. 点击圆点后，将该国家项目传给列表渲染逻辑。

职责划分：

- `maps/index.html`：图例结构和无障碍属性。
- `maps/styles.css`：图例、提示、断点布局、焦点状态和抽屉宽度。
- `maps/app.js`：自适应总览、抽屉避让、提示时序、搜索计数和键盘状态。
- `maps/data.js`：本轮保持不变。

## 7. 错误与边界处理

- 保留现有“无项目数据”提示，以及 `data.js` 或 Leaflet 脚本未加载时的“地图资源未加载”提示。
- 国家没有 `countryConfig` 时不伪造位置；现有项目总数也不能因此被静默改写。
- 地图瓦片加载失败时不切换到未经确认的新底图，也不新增瓦片级兜底；这属于后续可靠性优化范围。
- 动画遵循 `prefers-reduced-motion`。
- 抽屉打开、关闭和窗口调整期间，不允许出现 JavaScript 错误或重复动画。

## 8. 验证与验收标准

### 8.1 静态检查

- 页面中不存在“点位为国家级聚合，不代表项目坐标”。
- 页面中仍存在“圆点数字代表项目数量”。
- 移动端样式不会隐藏保留的图例文字。
- `maps/data.js` 无改动。
- CSS、JS 缓存版本号同步更新。

### 8.2 浏览器验收尺寸

至少验证：

- 1920 × 430：对应用户提供的宽嵌入截图比例。
- 960 × 520：中等嵌入。
- 760 × 520：响应式边界。
- 390 × 640：移动端回归。

每个尺寸检查：标题、状态、缩放控件、三个国家圆点、图例和总览按钮无重叠；抽屉可打开、搜索、展开项目并关闭。

### 8.3 功能验收

- 总数保持 51；马来西亚 27、文莱 23、泰国 1。
- 宽嵌入中东南亚点位成为视觉主体，三个圆点仍完整可见。
- 打开抽屉后选中圆点不被右侧抽屉遮挡。
- 搜索摘要显示匹配数量。
- `Esc` 可关闭抽屉，项目行的展开状态可被辅助技术读取。
- 无控制台错误。

## 9. 预览与发布流程

1. 在本地分支实现并运行静态检查。
2. 用本地 HTTP 服务生成上述尺寸的预览和截图。
3. 向用户展示本地预览，并提供简道云嵌入高度建议；宽嵌入建议优先从不低于约 520px 的高度开始试验。
4. 用户确认预览后，才更新 GitHub `main/maps/`。
5. 发布时同步更新 HTML 中的 CSS/JS 缓存版本号和正式链接版本参数。

- 对应断点的验证结果和截图。
